# PR作成前の事前チェックリスト

このリストは、PR本文を作成する前にプロジェクトルールを満たしているか確認するための項目です。
Skillはこれらの項目を確認し、「事前チェック結果」としてユーザーに報告してください。

## 1. テンプレートの確認
* [ ] `.github/pull_request_template.md` が存在する場合は、そのフォーマットを優先して読み込んでいるか？

## 2. プロジェクトルールの確認
* [ ] `README.md` に記載されている貢献ガイドラインやローカルルールに反していないか？
* [ ] `AGENTS.md` に記載されているコーディング規約や制約事項に違反していないか？
* [ ] `docs/` 配下に関連する設計ドキュメントやルールがある場合、それに従っているか？

## 3. テストと品質
* [ ] テストコードの追加漏れはないか？（既存のテスト方針に照らし合わせて確認）
* [ ] 実行していないテストを「実行した」と書いていないか？（未実行の場合は「未実行」と書くこと）

## 4. 記載内容の正確性
* [ ] 確認できていない影響範囲を勝手に推測していないか？（未確認の場合は「未確認」と書くこと）
* [ ] ルールとして明示されていないレビュアーやラベルを勝手に設定（提案）していないか？
