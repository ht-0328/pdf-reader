# 実行計画作成アシスタント (Execution Plan Writer)

このSkillは、ユーザーが達成したい目的に対して、必要な作業と順序を整理した実行計画（Markdownファイル）を作成します。
すべてが決まるまで計画作成を止める必要はありません。未決定事項を残した暫定計画を作成できます。

以下のフェーズに従って順次実行してください。
各フェーズでは、ファイル閲覧ツールを使用して指定されたテンプレートを読み込み、その指示に従ってください。

---

## 実行フロー

### Phase 1: 調査とヒアリング
ユーザーから依頼を受けたら、まずこのフェーズを実行します。

1. `templates/phase1-hearing.md` を読み込んでください。
2. 目的と期待する結果を理解し、必要な場合だけ関連情報を調査します。
3. 目的や対象が分からず、暫定計画すら作れない場合だけ重要な質問を最大3問出力し、**ここで停止してユーザーの回答を待機します。**
4. 回答がなくても暫定計画を作れる場合は、質問せずにそのまま Phase 2 へ進んでください。

### Phase 2: 計画作成
Phase 1 で質問が不要だった場合、またはユーザーの回答を得た場合に実行します。

1. `templates/phase2-planning.md` を読み込んでください。
2. 必要な作業と順序を整理し、未決定事項を明示した実行計画を作成します。
3. 保存前に計画を自己確認し、問題があれば一度修正してください。
4. 最終的な実行計画をMarkdownファイルへ保存します。
5. 保存後、保存先のファイルパスを簡潔に報告して完了とします。
