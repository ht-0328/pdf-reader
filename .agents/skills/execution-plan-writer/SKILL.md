# 実行計画作成アシスタント (Execution Plan Writer)

このSkillは、ユーザーが達成したい目的に対して、必要な作業と順序を整理した実行計画（Markdownファイル）を作成します。
また、既存の実行計画の確認・修正・更新にも対応します。
すべてが決まるまで計画作成を止める必要はありません。未決定事項を残した暫定計画を作成できます。

---

## モード判定

ユーザーの入力内容から、次のどちらのモードで処理するかを判定してください。

### 新規作成モード

既存の実行計画が提示されておらず、新しい計画の作成を求められている場合。

### 更新モード

既存の実行計画を確認・修正・更新する依頼の場合。次のような依頼が該当します。

- この計画を修正して
- この計画に変なところがないか確認して
- この内容を反映して計画を更新して
- 計画の抜け漏れを確認して
- 実行できる計画に直して
- 計画ドキュメントを更新して
- 問題点だけ教えて
- レビューだけして

---

## 実行フロー

### 新規作成モード

以下のフェーズに従って順次実行してください。
各フェーズでは、ファイル閲覧ツールを使用して指定されたテンプレートを読み込み、その指示に従ってください。

#### Phase 1: 調査とヒアリング
ユーザーから依頼を受けたら、まずこのフェーズを実行します。

1. `templates/phase1-hearing.md` を読み込んでください。
2. 目的と期待する結果を理解し、必要な場合だけ関連情報を調査します。
3. 目的や対象が分からず、暫定計画すら作れない場合だけ重要な質問を最大3問出力し、**ここで停止してユーザーの回答を待機します。**
4. 回答がなくても暫定計画を作れる場合は、質問せずにそのまま方針提示へ進んでください。

#### 方針提示と承認
Phase 1 で質問が不要だった場合、またはユーザーの回答を得た場合に実行します。

1. `templates/phase2-planning.md` の「方針提示と承認」セクションに従い、計画の作成方針をユーザーへ提示してください。
2. **ユーザーの承認を得るまで、計画の作成・ファイル保存を行わないでください。**
3. ユーザーが追加条件や修正を伝えた場合は、方針を修正して再度提示し、改めて承認を得てください。

#### Phase 2: 計画作成
ユーザーの承認を得た後に実行します。

1. `templates/phase2-planning.md` を読み込んでください。
2. 承認された方針に基づき、必要な作業と順序を整理し、未決定事項を明示した実行計画を作成します。
3. 保存前に計画を自己確認し、問題があれば一度修正してください。
4. 最終的な実行計画をMarkdownファイルへ保存します。
5. 保存後、保存先のファイルパスを簡潔に報告して完了とします。

### 更新モード

更新モードと判定した場合は、以下のフェーズを実行してください。

#### Phase U: 既存計画の確認・更新
1. `templates/phase-update.md` を読み込んでください。
2. テンプレートの指示に従い、更新対象の特定、確認、変更方針の提示と承認、修正、出力を行ってください。
3. 計画の品質ルール（未決定事項の扱い、形式、自己確認など）は `templates/phase2-planning.md` に定義されています。更新後の計画にも同じルールを適用してください。
4. レビューのみの依頼（ファイルを変更しない場合）は、承認確認は不要です。
